Learning Progression Design

How Play Teaches
Prompt Engineering.

Prompt Quest uses a 3-stage progression across 7 fantasy zones to teach children ages 8–10 how to write effective prompts — without ever using the word "prompt." Every stage introduces a new skill through play, not instruction.

The Core Thesis

Why mad-libs teach prompting better than textboxes
Key Insight

Prompt engineering is the skill of being specific. Children learn specificity best when they can see the consequences of being vague versus precise — not by reading rules about it.

Prompt Quest replaces the blank textbox (overwhelming for kids) with structured fill-in-the-blank templates. Each slot teaches one dimension of specificity: WHO does it, WHAT they do, and HOW they do it. By constraining the input space, we make the relationship between input precision and output quality visible and immediate.

The Prompting Parallel

These three stages mirror the three skills that make adults effective at prompting AI:

1. Give the AI a clear task — who is involved, what should happen, what's the context
2. Add constraints and modifiers — tone, scale, urgency, style parameters
3. Chain concepts together — combine techniques for emergent behavior, multi-step prompts

Stage 1: Structure

Learning to specify WHO, WHAT, and WHERE
1

The Skill: Prompt Structure

A vague request produces a vague result. Stage 1 teaches the most fundamental prompting skill: breaking a request into concrete components. Instead of "plan a party," the child specifies the food, the entertainment, and the vibe — and sees how each choice changes the outcome.

How It Works

The child fills in 3 slots in a sentence template. Each slot has 6 curated options (concrete nouns and adjectives, never abstract concepts). The combination triggers one of 30–36 pre-built vignettes with unique animations.

✓ A "prompt" has parts (WHO, WHAT, WHERE) — not just one blob of text
✓ Some combinations work better than others — alignment between components matters
✓ Even "wrong" answers are fun — failure is safe and entertaining

Stage 2: Precision

Learning that modifiers change everything
2

The Skill: Prompt Modifiers

Stage 2 keeps the base structure from Stage 1 but adds modifier slots — adjectives that control scale, mood, intensity, or approach. The child learns that adding "tiny" versus "MEGA" to the same base prompt produces wildly different results. This is the core of prompt engineering: constraints shape output.

How It Works

The template expands from 3 to 5 slots. The new slots are modifier dimensions: SIZE, MOOD, URGENCY, INTENSITY, or APPROACH (varying by zone). After each play, a Vague Comparison panel shows what would happen with a generic prompt versus the child's specific one.

Plan a SIZE party with FOOD and ENTERTAINMENT for a VIBE birthday where the skeleton feels MOOD
Purple slots = new modifier slots added in Stage 2
The Vague Comparison — Stage 2's Secret Weapon

After every vignette, the feedback card shows a side-by-side comparison: "What if you just said something vague?" This makes the value of specificity visible.

Vague Prompt

"Have a party"

A party happens... but how big? What mood? Nobody knows!

Your Specific Prompt

"Plan a TINY party with cake where the skeleton feels SHY"

A pocket-sized party with a tiny cake and a blushing skeleton!

This is how professional prompt engineers think: "If I don't specify the tone, the AI will guess. If I say 'formal' vs. 'casual,' I control the output." Stage 2 teaches this instinct through contrast, not lecture.

What the Child Learns (Implicitly)
✓ Modifiers are levers — the same base prompt + different modifier = completely different output
✓ Vague = unpredictable — without modifiers, the AI (or game) guesses for you
✓ Scale matters — "tiny" and "MEGA" aren't just adjectives, they're control parameters
✓ Emotional tone shapes outcome — "excited" vs "shy" changes behavior, not just label

Stage 3: Composition

Learning that combining ideas creates something new
3

The Skill: Combinatorial Thinking

Stage 3 shifts the mental model entirely. Instead of filling in parts of a single request, the child now combines two concepts to discover emergent behaviors. Magic + Fireworks doesn't produce "magic AND fireworks" — it produces Spell Explosions, something neither concept alone could create.

This mirrors advanced prompting: chaining techniques (e.g., "use chain-of-thought AND few-shot examples") creates capabilities that neither technique provides alone.

How It Works

The template gives two combo slots from the same category plus two context slots. The child must discover 4 secret combos to complete the stage. Progressive hints guide exploration without giving answers immediately.

Combine ACTIVITY 1 with ACTIVITY 2 in a SPIRIT way at LOCATION
The Progressive Hint System

When a child gets stuck, hints escalate from curiosity-provoking to explicit — modeling how to refine a search strategy:

?
Hint Level 1: Provoke Curiosity
"What happens when magic meets explosions?"
No answer given — just a question to spark thinking
!
Hint Level 2: Narrow the Search
"Magic tricks that go BOOM..."
Directional guidance without the explicit answer
✓
Hint Level 3: Direct Answer
"Try Magic + Fireworks = Spell Explosions!"
Prevents frustration — discovering is always better than being stuck

This mirrors iterative prompting: start broad, refine based on output, get more specific until you find what works.

What the Child Learns (Implicitly)
✓ Combining concepts creates emergent results — A+B produces something C that neither alone could
✓ Exploration is systematic — not random guessing, but hypothesis-driven testing
✓ Iteration improves results — hints model the "try, refine, try again" loop
✓ Discovery is rewarding — finding hidden combos creates genuine delight

The Feedback Loop

How every play reinforces learning without feeling like a lesson
The Four-Part Feedback Card

After every vignette, the child sees a feedback card with four components designed to reinforce without lecturing:

1
Score Badge
Emotional framing, not judgment
Four tiers: PERFECT (celebration), FUNNY FAIL (comedy), CHAOS (excitement), PARTIAL (encouragement). There is no "wrong" — only different flavors of fun. The badge uses emoji + color to communicate tone instantly.
2
Narration + Filled Sentence
Mirror back what the child built
Shows the child's completed prompt in context: "You planned a TINY party with Cake and Magic Show." This reinforces the connection between their choices and the outcome.
3
Skill Badge + Concrete Tip
One skill, one actionable suggestion
A purple pill shows the skill being practiced ("Specificity" or "Combo Thinking"). The tip is always concrete and actionable: "SIZE changes EVERYTHING. Try GIANT for mega chaos!" Never abstract metacognition.
4
Stage-Specific Feature
Adapts to the current learning goal
Stage 1: discovery counter. Stage 2: vague comparison panel. Stage 3: combo progress bar. Each stage adds the feedback mechanism that best reinforces its specific skill.
Feedback that works
✓ "Try GIANT for mega chaos!"
✓ "SIZE changes everything"
✓ "What if you combine these two?"
✓ "You nailed WHO, WHAT, and HOW"

The 7 Zones

Same progression framework, different contexts — because transfer requires variety
Why 7 Zones?

Learning research shows that children need to practice a skill in multiple contexts before it transfers to new situations. A child who learns specificity only through party-planning won't instinctively apply it to space missions. Seven thematically diverse zones ensure the prompting skills generalize — the child learns "being specific works everywhere," not "being specific works for parties."

💀

Scaffolding Design

How complexity increases while anxiety stays low
Principle: Additive Complexity

Each stage adds to the previous one rather than replacing it. Stage 2 keeps all Stage 1 slots and adds modifier slots. The child's existing knowledge is never invalidated — it's extended.

Safety Nets

Every stage has mechanisms to prevent frustration:

✓ Default vignettes — every tag combo resolves to something, even if it's a wildcard fallback
✓ No wrong answers — "funny fail" is scored, animated, and celebrated
✓ Discovery counter — shows progress even when not "winning"
✓ Progressive hints (Stage 3) — scaffolded help that increases in specificity
✓ Persistence — progress saves to localStorage, so quitting doesn't lose work
The Failure Spectrum

Traditional educational games have binary pass/fail. Prompt Quest uses a 4-tier spectrum where every tier is entertaining:

PERFECT
Aligned components, spectacular result. The child sees what a well-crafted prompt achieves.
PARTIAL
Some components align, some don't. Shows that prompts can "sort of" work — encouraging refinement.
FUNNY FAIL
Misaligned components produce comedy. The child wants to see what goes wrong. Failure becomes a feature.
CHAOS
Maximum spectacle, maximum fun. Chaotic combos create the most dramatic animations. Kids chase chaos on purpose.

The Graduation Moment

A child who completes all 3 stages across multiple zones has internalized these instincts:

1. "I should be specific" — from seeing vague vs. specific outcomes hundreds of times
2. "Details change everything" — from watching modifiers transform the same base scene
3. "I can combine ideas" — from discovering that A+B creates C, not just A and B
4. "I should experiment" — from the reward loop of discovery and progressive hints

These four instincts are exactly what make an adult effective at prompting AI. The child has learned prompt engineering without ever being told they were learning it.
